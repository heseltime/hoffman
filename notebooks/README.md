# README

## Setup 1 - Environment-Management

.venv-environment based on local requirements.txt recommended, VS code allows installing the `genai-stack/requirements.txt` in hoffman too: ultimately any tests done in notebooks need to be brought back to the genai-stack (and Docker system) if they are integrated into the hoffman-stack. Approach mbe: keep local requirements.txt minimal and bring to `genai-stack/requirements.txt` when copying code over.

## Setup 2 - Alt Approach (Prefer)

One Workspace Environment for hoffman: notebooks and `genai-src` that can be pulled into the GenAI stack (`genai-stack`). Docker runs its own environments independently. This approach requires being careful with manageing the requirements.txt in the root directory: testwise versioning and so on might make sense.

### Add-on 

Do add a local requirements.txt for notebooks: experiment here, move out code needed for Docker/pipeline to `genai-src` and hoffman/root-level requirements.txt.
